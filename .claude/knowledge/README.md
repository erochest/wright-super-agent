# Knowledge

This directory contains domain reference material — the "textbook" layer of the two-layer knowledge architecture.

## Two-Layer Model

| Layer | Location | Purpose | Access |
|-------|----------|---------|--------|
| **Domain Knowledge** | `.claude/knowledge/` (here) | "How should I think about X?" | Loaded at startup |
| **Institutional Memory** | `docs/solutions/` | "What happened when we tried X?" | Searched on demand |

## How to Use This Directory

Add curated reference files here as you develop domain expertise. These files should be:
- **Stable** — updated when methodology evolves, not after every task
- **Curated** — human-reviewed, not auto-generated
- **Focused** — each file covers one domain topic

Start empty. Build knowledge through operation, not pre-seeding.
